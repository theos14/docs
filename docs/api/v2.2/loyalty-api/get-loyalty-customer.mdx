The Loyalty API is currently in beta.

Please contact our Support Team via our [JIRA Service Desk](https://primerapi.atlassian.net/servicedesk/customer/portal/11) for access to the Beta. If you don’t have access, please contact your account administrator for assistance.

Not yet integrated with Primer and interested in learning more? [Contact us](https://primer.io/contact-us/).

Create an API key with the `loyalty-transactions:read` scopes.

Get a loyalty customer by ID and connection ID. This includes the customer’s points balance.

The `connectionId` is the unique Primer identifier for your loyalty provider connection. Primer will provide this once the loyalty provider connection is created.

The `customerId` refers to the customer identifier on the loyalty provider’s side.

## Path Parameters
<ParamField path="id" type="string" required> The unique identifier for the customer on the loyalty provider. </ParamField>
## Query Parameters
<ParamField query="connectionId" type="string" required> ID of the Primer connection configuration. </ParamField>
## Response
The endpoint returns an object with the following properties:

<ResponseField name="id" type="string" optional> The unique identifier for the customer on the loyalty provider. </ResponseField> <ResponseField name="connectionId" type="string" optional> ID of the Primer connection configuration. </ResponseField> <ResponseField name="connectionName" type="string" optional> Name of the loyalty provider. </ResponseField> <ResponseField name="balance" type="integer" optional> The loyalty customer's points balance. </ResponseField>

## Primer's currency policy

The `amount` field in the Primer APIs is in minor units i.e. the smallest unit of a currency.

A currency can either have 0, 2 or 3 minor units. For example:

* JPY has 0 minor units, so to create a payment for 1000 JPY, `amount` should equal `1000`
* USD has 2 minor units, so to create a payment for 10.00 USD, `amount` should equal `1000`
* IQD has 3 minor units, so to create a payment for 1.000 IQD, `amount` should equal `1000`

## Supported currencies

**The following currencies are disabled by default and require your Primer account to be "_gated in_" in order to use them:**

BHD, BIF, CLP, CVE, GNF, HUF, IDR, INR, ISK, KHR, KMF, MMK, PYG, RWF, SAR, UGX, VED, VES, VND, XOF, XPF

If you try to use one of these currencies without being gated in, you will receive the following API validation error message:

```json JSON
{
    "error": {
        "errorId": "DisabledCurrencyError",
        "description": "HUF is a disabled currency",
        "recoverySuggestion": "To enable, please speak to your Customer Success Manager or our Service Desk https://primerapi.atlassian.net/servicedesk/customer/portal/11.",
        "diagnosticsId": "9942005157443145021"
    }
}
```

To be gated in, speak to your Customer Success Manager or raise a ticket on our [JIRA Service Desk](https://primerapi.atlassian.net/servicedesk/customer/portal/11).  If you don’t have access, please contact your account administrator for assistance.
